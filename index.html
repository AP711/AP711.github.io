Bug & Requirements Intelligence Agent

Personal R&D Project · 2025–2026
  • Built an AI-powered developer productivity tool that automatically generates a structured intelligence report for any Jira bug ticket — replacing 60–90 minutes of manual investigation with a sub-2-minute automated analysis.
  • The agent connects to Jira and Confluence, retrieves historical bug data, requirement documentation, and linked pages, then produces a confidence-scored analysis covering: similar defect detection, requirement coverage verdicts (Covered / Partially Covered / Not Documented), blast radius assessment, and recurring risk pattern identification.
  • Engineered a multi-step stateful pipeline using LangGraph, with discrete nodes for data gathering (Jira REST API, JQL, Confluence), ADF-to-text conversion, LLM-based analysis, and HTML report rendering via Jinja2.
  • Implemented a deterministic confidence scoring system (0–100%) computed before the LLM call — making results reproducible, explainable, and auditable.
  • Designed a continuous feedback loop where engineer ratings are distilled into component-level insights injected into future analyses, improving accuracy over time at a fixed token cost.
  • Built with a clear versioning roadmap: V1 (CLI tool, current), V2 (VS Code extension), V3 (FastAPI + React web application with OAuth 2.0).
